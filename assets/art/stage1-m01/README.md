# M01 Art Slice Assets

Status: first paper-backed runtime candidate set imported through Cocos.

This directory is reserved for the first M01 art-calibrated slice. It should contain only assets that replace or inform the highest-signal greybox pieces:

- gear-star body
- nine-slot classification tray
- memory fragments
- color filters
- ToolCard thumbnail basis

Use the active references in `docs/design/style-references/README.md`, with `docs/design/style-references/2026-04-22-unified-handdrawn-style-anchor.png` as the primary game-style reference. The UI/card references are secondary and should only inform panel or ToolCard language. Do not use the older Stage 1 scene batch as the production style target.

Generated review candidates still live outside the Cocos `assets/` tree at `docs/design/generated-m01-art-slices/`. The PNGs in this directory are the first accepted paper-backed runtime candidates cropped from `m01-runtime-sprite-sheet-candidate-v2.png`; each has a matching Cocos `.png.meta`.

## Acceptance

- Clear charcoal ink contours.
- Low-saturation clay red, dusty blue-gray, muted ochre, rice-gray, and gray-bronze palette.
- No baked-in readable text.
- Soft low-contrast paper grain and natural watercolor pooling are acceptable.
- No high-contrast white speckle, scattered white flecks, salt-and-pepper noise, or distressed paint chips inside mechanical color masses.
- Slot and fragment identities stay readable at gameplay scale.
- Greybox must remain playable when art assets are disabled.

## Import Rule

Do not place PNGs in this directory unless they are accepted runtime candidates or source assets ready for Cocos import. Every accepted PNG in this directory must have a matching Cocos `.png.meta` generated by the editor before the asset is considered ready for integration.

These first candidates remain paper-backed, not transparent isolated sprites. They are suitable for visual scale checks and early Cocos import validation; final gameplay replacement may still need per-piece transparent crops or an atlas/sprite-frame slicing pass.

## Runtime Loading

These files are not under `assets/resources`, so they are not directly loadable through `resources.load`. Use their `db://assets/...` URLs for editor/MCP import checks only. Gameplay integration should use serialized scene references, UUID-backed references, a later `assets/resources` placement, or an atlas/sprite-frame strategy.
