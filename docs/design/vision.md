# 《流辉美慧号：星图守护者》— 产品愿景

---

## 这是什么

一个披着游戏外衣的益智启蒙工具。它的目的不是娱乐，而是帮助人丰富思考、扩建思维能力。游戏只是载体，认知成长才是内核。

## 给谁用

对思维智能成长感兴趣的成人及青年玩家（16+）。

不是"6-12岁儿童教育游戏"——那个定位会限制认知深度的上限。一个对哥德尔不完备定理的隐喻、一次关于决定论与自由意志的体验，这些内容不应该因为"目标用户是小孩"而被稀释。

童话意象仍然保留：莱米、小兔子、星星、花园、齿轮都可以温柔、直觉、可亲近。但它们服务的是成人也能咀嚼的认知隐喻，不服务低龄化讲解。

## 核心设计原则

### 诗意美感，而非组件摆放

每个关卡应该是一件有意境的作品。交互本身要有美学体验，不只是功能性的操作。

"拖拽砝码到天平上达到平衡"——这是操作。
"两颗星星之间的光带像呼吸一样起伏，你调整的不是重量，而是它们之间的共鸣"——这是体验。

### 每一关是一个隐喻

关卡的核心不是"解一道题"，而是"经历一个隐喻"。交互是手段，领悟是目的，美感是贯穿始终的语言。

### 思维的提升分阶段递进

50关分为5个阶段，有从易到难的递增感。具体编排自由，但整体弧线必须让玩家感受到自己的思维在生长。

## 不变的要素

- **莱米**：主角小兔子，驾驶流辉美慧号穿越星海
- **修复星星**：每颗星星的故障对应一种认知体验，修复过程就是思维训练
- **智慧结晶**：每关完成后获得一句凝练的感悟
- **Arrog 式统一手绘墨线 + 低饱和淡彩 + 功能性简约机械**的美术风格。当前只以 `docs/design/style-references/` 中三张主动参考图为准：M01 手绘锚点 + 两张游戏界面/面板参考图。
